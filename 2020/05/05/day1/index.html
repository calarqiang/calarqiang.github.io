1. 关键字和保留字

关键字：

定义：被java语言赋予了特殊的含义，用作专门用途的的字符串，所有关键字均为小写。比如class、public、

保留字：

定义：指现有java中存在的但是未使用，可能作为后续的版本去使用。

比如：goto、const等。避免命名标识符去使用保留字！

2. 标识符

定义：凡是可以自己取名字的均为标识符。

  • 类名
  • 接口名
  • 变量名
  • 方法名

标识符命名规则：

  1. 不要使用关键字或者保留字
  2. 不要以数字开头
  3. 只能使用字母、数字、下划线和$(美元符)
  4. 严格区分大小写
  5. 不限制长度

标识符命名规范：

  1. 如果是包名，含有多个单词组成，则每个单词都小写。
    • xxxyyyzzz
  2. 如果是类名、接口名：含有多个单词时，每个单词的首字母都大写。 (大驼峰命名法)
    • XxxYyyZzz
  3. 如果是变量名、方法名：含有多个单词时，第一个单词首字母小写，其余单词首字母大写。 (小驼峰命名法)
    • xxxYyyZzz
  4. 如果是常量名：含有多个单词时，每个单词均大写，并用_(下划线)连接起来。
    • XXX_YYY_ZZZ

关于java内部的编码问题：java内部采用unicode编码，因此变量命名时，可以采用汉字的方式：int 数字=12,但是不建议使用！

3.1 变量按照数据类型划分

3.1.1 基本数据类型

基本数据类型包括：(四类八种)

  • 数值型
    • 整数类型
      • byte (1字节)
      • short （2字节）
      • int （4字节）
      • long （8字节）
    • 浮点类型
      • float （4字节）
      • double （8字节）
  • 字符型 (char) （2字节）
  • 布尔型 (boolean)

注意：在java中每个数据类型所占有的字节数和具体的操作系统和平台无关，任何操作系统或者平台都是上面的字节数！

  1. 关于变量的声明和定义：

    变量定义：int myAge = 12; 此时不仅声明了变量，还给予其赋初值，可以使用

    变量声明：int myAge; 此时只是声明了一个变量，并没有赋值，不可用

    变量赋值： myAge = 13;

    变量定义=变量声明+变量赋值

    变量必须声明后，才可以赋值使用。

  1. 关于java中变量的作用域问题：

    1. 在java中是存在作用域的，即在一对{}内部就是一个作用域，变量的声明定义只有在该作用域中有效，当出了该作用域，就访问不了。

    2. 不可以在同一个作用域中声明同一个变量。

整型

四种：byte、short、int、long

几个结论：

  1. 默认整型常量不指明类型时，均为int,比如34就是一个int类型常量

  2. byte范围为-128~127

  3. 声明long类型变量时，如果想让其实long类型的，那么后面必须加一个l或者L

  • long a=1234L/1234l
  • 如果此时不写后缀，那么默认第二点可知，赋值给long变量的是一个int类型的，此时会发生隐式类型转换（后面所说的变量类型提升）,此时也不错！只是语义可能会产生误解而已。
  1. 在开发中，不管数字是多大，一般都用int类型

浮点型

float、double

几个结论：

  1. 默认浮点类型常量不指明类型时，均为double,比如3.4就是一个double类型常量
  2. 声明float类型变量时，后缀必须加f或者F
    • 因为默认赋值的浮点数为double类型，如果不加后缀的话，会导致大的数据类型转换为小的数据类型，由于没有加上强制类型转换符（），此时会发生异常，报错。
  3. 在开发中，一般使用double类型变量。

字符型

char

几个结论：

  1. 声明字符时，要用'',内部必须写一个字符，可以是中文也可以是英文。
    • 不能写成char c=''里面啥都没有，这是错误的写法，只有字符串能为空，字符不可以。
  2. 字符的不同赋值方式：
    • 普通：char c='a'
    • 转义字符： char d='\n'
    • 直接使用unicode: char e='\u0043' (C 67) 里面的是十六进制

关于字符集和字符编码：

我们在编写一个文档的时候，要指定编码格式，此时该文档的内容均会按照指定的编码格式转换为二进制，当读取时，会将二进制按照读取的编码格式读取出来，如果存入的编码格式和读取的编码格式不一致，则会导致乱码的出现！

关于gbk编码和ANSI编码区别：

  • ANSI其实是ascll编码，用一个字节，128（0~127）种情况来表示所有的英文字母、数字、特殊的字符，最初就是在美国推行，最初的编码。
  • gbk编码是中国的汉字编码，用两个字节表示大部分的汉字和所有ascll码表示的内容。它的前0~127和ascll码一致，即包含ascll码，后面的开始就表示汉字或者其他的特殊字符。

布尔型

只有两种取值：true或者false

基本数据类型之间的运算关系

前提：只讨论除了boolean类型之间的其余基本数据类型。

  1. 两个概念：

    • 自动类型提升（隐式类型转换，内部自动完成）

      当容量小的数据类型和容量大的数据类型之间进行运算时，结果会自动地转换为容量大的数据类型。

      注意： ** **容量大小并非数据类型所占有的字节数多少，而是表示数据范围的大小。

      byte、short、char–>int–>long–>float–>double

      byte、short、char两两之间进行运算或者同一类型进行运算，都会自动转换为int类型

      疑问？

      1. 为什么可以直接byte/short/char=12整型常数（int类型）

        ​ 按照之前的结论，这样必须要加强制类型转换符，为啥可以通过编译不报错？

      2. 关于编译器的常量优化：（只对byte、short、char有效）

        short/byte/char=2+3;这样不报错，为啥？

    原因：因为赋值表达式的右边全是常量，没有变量，在编译阶段即将其优化为Short a=13,由注意事项一可知，它右边未超过左边范围，因此可行.
    如果一边只要含有变量，都会遵循上述的，会转换为int类型的，再赋值就会报错。
    short a=2;
    short b=4;
    short c=a+b; //错，int无法转换为short

    short x=12;
    short y=x+12; //错，右边不是全是常量，不会进行编译器常量优化。

  1. 关于原码、反码、补码

    正数的原码、反码、补码均为同一个！

    负数的原码为正数的原码将符号位由0变为1,反码为在原码的基础上保持符号位不变，其它均取反，补码则是在反码的基础上+1

    一般写二进制时，不管是正数还是负数，只要没有特殊说明，都是补码形式。

    • 解释：

      分析：

      首先a为正数，在内存中由32个bit构成，前24个高位为0,(其中第一位为符号位),后面8位为10000000,此时表示正好是+128

      当下面进行转换时，是将int类型转换为byte型，由于byte只有八位，因此该八位为10000000，前面还有24位都被截取舍弃掉了，因为容不下。因此b在内存中的表示就为10000000,由于内存中表示的都是补码，因此正好-128的补码是10000000,所以就会输出-128

string类型表示字符串类型，它可以为一个空字符串，比如string s="",但是字符类型是不可以有空字符的，char c=''(错误)

4. 运算符

  • 算术运算符

    • +、-、*、/、%

      • 关于除法：

        int a=12,b=5; int result=a/b (结果为2,因为int/int仍为int)

      • 关于取余运算：

        • 当两个数值进行取余运算时，结果的符号是看被模数的。

    • 关于自增运算符：

      • 前自增/自减

        • ++a,–a
          • 前自增和自减，是先将数自增/自减后，然后再进行其他的运算（比如赋值、相加等）

      • 后自增/自减

        • a++,a–

          • 后自增和自减，是先进行运算（赋值、加减等），然后再进行自增1或者自减1

        • 特别注意：

          当变量类型为byte、short、char时，自增或者自减不会改变数据类型，即不会转换为int类型，但是这样写会转换 byte a=12; a=a+1; //编译不通过，因为会转换为int类型

> 题目：
>
> int n=10;
>
> n+=(n++)+(++n)   //n的值为32
>
> n=n+(n++)+(++n)  //10+10+12=32

  • 赋值运算符

    • 支持连续赋值：

      • int a,b; a=b=12;

      • 特别注意：

        关于+=、-=、*=、/=、%=等操作时，如果类型为byte、short、char时，不会转换为int类型

        eg:byte a=12;a+=3; //这样是正确的。如果写成：byte a=12;a=a+3; //编译不通过，因为转换为了int类型。

  • 比较运算符

  • 逻辑运算符

    &(逻辑与)、|(逻辑或)、！(逻辑非)、^(异或)

    &&(短路与)、||(短路或)

  • 关于异或运算：

    异或是相同为0，不同为1

    异或只能用于数值运算，不能用于布尔类型运算。

关于逻辑运算符和短路运算符的区别和联系：

  • 联系：当他们都进行布尔值的运算时，得出的结果是一致的。

  • 区别：

    1. 逻辑运算符除了可以两边是布尔值运算时，还可以是数字类型的运算，比如12&4;

    2. 短路运算符只能进行逻辑运算，无法进行数值类型计算。

    3. 当两边是布尔值运算时：

      • 逻辑运算符必须将运算符两边的都进行计算了之后才得出结论。

      • 而短路运算符是短路操作，如果是&&操作，那么得到一个为false后，就不会继续运算下去，如果是||的话，得到一个结果为true的话就不会继续运算下去。

  • 位运算符 （只能操作整型）

    • <<(逻辑左移)、>>(逻辑右移)、>>>(无符号右移)

      • 逻辑左移：

        逻辑左移是将二进制数据向左移动，末尾补0，正数可能会移成负数。

        ‘负数也可能移成正数。

        左移的本质是将数扩大了移动的2的多少倍。

      • 逻辑右移

        将二进制向右移动，左边按照该数原来的正负来补0或者1（正数补0负数补1）

        右移的本质是将数缩小了移动的2的多少倍。

      • 无符号右移

        将二进制数向右边移动，和逻辑右移不同的是，不管该数是正数还是负数，移动后前面统一补0.

  • 分支结构：

    对于if-else结构，可以只存在if,因此可以没有else结构。

    针对条件表达式：

    1. 如果多个条件表达式之间是“互斥关系”的话（即没有交集），此时可以颠倒if-else中的判断条件。
    2. 如果多个条件表达式中有交集的情况下，应根据实际需求，选择顺序。
    3. 如果多个条件表达式中是包含关系的话，通常情况下，将小的范围的条件表达式放在上面，大的放在下面，因为如果顺序颠倒，那么小的那个条件表达式就永远不会执行。

    switch-case结构：

    switch-case结构中的所有case是并列结构，因此和书写顺序没关系（前提是，加了break)

    在switch结构中，switch中的表达式只能是六种类型：byte、short、char、int、enum(枚举类型，jdk5才可以)、String类型（jdk7才可以）

    case后面只能是一个常量，而且注意要与Switch中的数据类型匹配。

    break关键字可省略可不省略，各有用处。

    default关键字也是可以省略可以不省略。

关于三元运算符和if-else和switch结构的选择

总体来说，这三种结构最适应性强的就是if-else结构，如果能用前两种结构完成的，都可以用if-else来实现。

  • 效率：如果能用其它两种完成的程序的话，就用其它两种，他们的效率比if-else效率略高。
  • 适应性：if-else适用性是最强的，基本上开发主要用它。
